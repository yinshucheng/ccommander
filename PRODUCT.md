# Product

## Register

product

## Users

同时开着多个 AI coding 会话（Claude Code / Codex 等）的开发者与操作者——可能是一个人，也可能是一个小团队。他们的共同点不是"人多"，而是**手里同时跑着多条并发的 agent**：一个在跑、两个等回复、一个卡在澄清上。

他们要的不是"管理更多 agent"，而是两件事：

- **提高自己的智力活动密度**——把每一分钟都花在思考、判断、决策上，而不是轮询状态。
- **不再在等待中分神**——机器在跑的时候，人不该被一个接一个的进度条牵着走、反复切窗口检查"谁又动了"。

他们使用 Commander 时的状态，是**坐下来逐条批阅**，而不是站在塔台四处张望。

## Product Purpose

Commander 是一个**被动观测、为你调度注意力**的指挥台。机器上正在跑的 AI 会话自动汇聚成卡片，按"谁最该被你看见"排序，最紧要的压在最上面，一道朱批，下一份自动呈上。

它不造工具、不派活、不接管你的会话——你照常用原生的 `claude`、`codex`。它只在旁边安静地观测，然后回答唯一一个问题：

> 此刻，我那么多正在跑的会话里，**到底该把注意力放在哪一个，为什么。**

它不让你跑得更快（那是 AI 的事）。它让你在 AI 替你跑的时候，**重新成为那个做判断的人，而不是盯着五个进度条的值班员**。

**成功的样子**：用户不再轮询、不再纠结"接下来看哪个"；处理完一条下一条流畅呈上；一段时间里他做了大量判断与决策，而几乎没有时间花在"找该看哪个"上。

## Brand Personality

**Sovereign · Calm · Decisive**（君临 · 沉静 · 果断）。

定位是"皇帝批阅奏章"：你是做判断的人，不是跑腿盯梢的人。

- **Voice / Tone**：克制、笃定、有尊严。陈述事实，不喧哗、不催促、不讨好。该提醒时一次说清，绝不靠红点和角标制造焦虑。
- **情绪目标**：让人感到**掌控**与**从容**——"这些它替我盯着，我只需逐条拍板"。安静的权威，而非花哨的活力。
- **对自己诚实**：连状态准确度都明说（hook ~95% / 纯扫描 ~70%），不假装无所不知。诚实本身是这个产品姿态的一部分。

## Anti-references

- **玩具 / 花哨的 AI 应用**：拒绝可爱吉祥物、满屏 emoji 装饰、什么都圆角到底、渐变文字。这是一件给认真工作用的严肃仪器，不是消费玩具。
- **繁忙的 SaaS 仪表盘**：拒绝 hero-metric 大数字卡、KPI 瓷砖、渐变点缀、千篇一律的卡片网格、图表噪音。它是一个**分诊/批阅**界面，不是数据分析后台。
- **加剧轮询的通知轰炸**：拒绝 Slack/Jira 式的角标狂轰、到处红点、未读计数。Commander 的存在就是为了**终结轮询**，它本身绝不能成为新的轮询源。
- **杂乱的 IDE / 终端复用墙**：拒绝它要取代的那种 cmux/tmux 满屏分屏的观感。**一次只呈现一个**，刻意为之。

## Design Principles

1. **替你等，替你排序，判断留给你。** 界面只做两件事：把状态盯准、把"该看哪个"算清。决定永远是用户的——UI 不替用户做选择，只让选择变得轻松。

2. **一次只呈现一个。** 像奏章一样，最该看的压在最上面，逐条呈上。抵抗"把全部信息一次铺满"的冲动；全局是为了排序，焦点永远落在 current 那一条上。

3. **安静即尊重。** 不喧哗、不催促、不靠焦虑驱动。动效克制、色彩克制、文案克制。任何"为了吸引注意"而加的元素，都与产品宗旨相悖。

4. **被动观测，不抢方向盘。** 设计语言要透出"我在旁边帮你看，不接管你的工作流"。它是用户工作流的旁观增益，不是新的主入口。

5. **对自己诚实。** 把不确定（状态准确度、扫描兜底的漏报）如实呈现，而不是用满格的确定感糊弄用户。诚实换来的是信任，信任才让用户敢把"该看哪个"交给它排。

## Accessibility & Inclusion

- **目标 WCAG AA**：正文对比度 ≥ 4.5:1，大字 ≥ 3:1。当前是暗色界面，注意 muted 文字别掉到对比度下限。
- **优先级不只靠颜色**：P0–P3 当前用红/琥珀/绿/蓝表达——必须**叠加非颜色编码**（图标 / 文字标签 / 形状 / 位置），照顾红绿色盲用户，不让色弱用户丢失关键的"谁最紧要"信息。
- **完整支持 `prefers-reduced-motion`**：每个动效都要有降级方案（淡入或瞬时切换），尤其"下一条自动呈上"的过渡。
- **键盘优先**：批阅是高频连续操作（Enter 完成 / S 跳过 / L 稍后 / D 移除），键盘可达性是一等公民，焦点态要清晰可见。
