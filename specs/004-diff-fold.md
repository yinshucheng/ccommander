# 004 — Diff 上下文折叠

- **状态**: proposed
- **优先级**: 中
- **创建**: 2026-06-10
- **依赖**: [001](001-structured-transcript.md)

## 背景 / 动机

当前 Edit 的 diff 视图把未变更的上下文行全部展开，大文件的小改动会被淹没在大段不变的代码里。

## 目标

diff 只显示变更行 ± N 行上下文，中间未变更段折叠为「… 省略 X 行 …」可点开。

### 非目标

- 不改 diff 算法（仍用 jsdiff `diffLines`）。

## 需求

- 作为用户，我希望一眼看到「改了什么」，而不是滚动一屏不变的代码。

## 验收标准

- [ ] 未变更段超过 2N 行时折叠为占位行，显示省略行数
- [ ] 点击占位行可展开该段
- [ ] 变更行及其上下 N 行始终可见（N 默认 3）
- [ ] 短文件（无可折叠段）行为不变

## 技术方案

- `src/client/parts.jsx` 的 `DiffPart`：遍历 `diffLines` 结果时，对 `c.added===false && c.removed===false` 的大段做首尾保留、中间折叠。
- 折叠占位行是受控展开（局部 useState）。

## 风险 / 待定

- 折叠后行号对应关系需保持清晰（当前未显示行号，是否要补行号待定）。
