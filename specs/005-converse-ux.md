# 005 — 续话不自动切卡片

- **状态**: proposed
- **优先级**: 中
- **创建**: 2026-06-10
- **依赖**: 无

## 背景 / 动机

在某张卡片里发起「续话」时，后端推送的状态更新可能把当前面板切到另一张卡片，打断正在进行的对话，体验割裂。（上个 session 遗留的已知问题。）

## 目标

用户正在某卡片续话/查看时，自动排序或推送不应抢走焦点。

### 非目标

- 不改 liveState 的优先级排序逻辑本身（仅改「是否因排序变化而切换激活卡片」）。

## 需求

- 作为用户，我在 A 卡片打字/读对话时，即使 B 卡片变成「等我」，面板也不要自动跳到 B（可给提示徽标，但不抢焦点）。

## 验收标准

- [ ] 用户有「活跃交互」（输入框聚焦 / 正在续话 / 刚滚动）时，推送更新不切换激活卡片
- [ ] 其它卡片状态变化以非侵入方式提示（角标/高亮）
- [ ] 用户空闲时恢复原有自动排序/置顶行为
- [ ] 续话流式回复期间绝不切走

## 技术方案

- 前端：引入「交互锁」状态（聚焦/sending/最近交互时间戳）；`SessionPanel`/上层在锁定时冻结 `active` 切换。
- 需确认当前自动切换发生在哪一层（轮询刷新？ws 推送？）后再定改点。

## 风险 / 待定

- 「活跃交互」的判定边界（多久算空闲）需调。
