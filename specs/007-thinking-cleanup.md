# 007 — thinking 重复字符清洗

- **状态**: proposed
- **优先级**: 低
- **创建**: 2026-06-10
- **依赖**: [001](001-structured-transcript.md)

## 背景 / 动机

部分 thinking 文本出现字符级重复（如「用户想用户想测试…」），疑似 ccr / 某些 provider 的流式输出问题。这是**源数据层面**的瑕疵，不是渲染 bug，但影响阅读。

## 目标

在不损伤正常内容的前提下，尽量修复 thinking（及可能同样受影响的 text）的重复字符。

### 非目标

- 不在 Commander 侧解决 ccr/provider 的根因（那在代理层）。
- 不做激进改写，宁可漏修也不误删。

## 需求

- 作为用户，我希望思考块可读，不被重复字符干扰。

## 验收标准

- [ ] 能识别并消除「相邻片段逐字重复」模式
- [ ] 正常的重复表达（如「很好很好」「啦啦啦」）不被误删
- [ ] 仅作用于 thinking（评估是否扩展到 text）
- [ ] 有开关或可回退（怀疑误伤时能关掉）

## 技术方案

- 候选：在 `src/server/transcript.js` 的清洗阶段加启发式去重（检测交错重复 `ABAB→AB`、连续重复段）。
- 先采样真实数据统计重复模式，再定规则，避免误伤。

## 风险 / 待定

- 启发式易误删，需保守；是否值得做取决于 ccr 侧能否根治。
